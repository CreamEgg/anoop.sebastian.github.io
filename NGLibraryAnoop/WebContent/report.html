Data Storage

The Database Tier of this application is managed using the Hibernate ORM Framework. The data model for the database is mapped from objects and the data model generated for each objects are displayed below.

The Book Object

The Book object stores the data about the books stored in the library. Every books contains an ISBN, a title, an author, a category, a description, quantity (10 by default), a cover image path, a rent date (initialised null) and an expiry date (initialised null)

User's books stored as a Collection

The rentedbooks_sebasta2 table stores the rented books by all the users. It has a many to many relationship with the book and user objects. The rentedbooks_sebasta2 table stores the userid of the user and the isbn of the book. When a books is returned by the user, the entry is removed from this table.

Project Description

The Next Generation Library System features a library system where anyone can lookup the library information and the books that are available.

Users need to register in order to be able to rent books from the library. Once registered, users can log in and rent as many books as they want. Each book can be rented for five days and must be returned by the the expiry date. Once the user logs in they are able to see the books they have rented, the books they have returned and the due dates of the books that have currently rented.
Librarians manage the library. Librarians requires their Librarian ID in order to register themselves as an librarian. Librarians can see all the activities in the library, view currently rented books by the users, all registered users' information. Librarians can assign a book to a user and they can also modify an expiry date for a user's book.

Features

Responsive design

The front-end framework Boostrap is used to develop a responsive website that works across different devices

Fixed Title Bar

The Title Bar remains on top no matter wherever the user browses to in the website. This enables the user to access all the main links from any page.

AJAX enables Login/Register system with serverside validation.

The login and registration is handled using AJAX and if the credentials entered are wrong the user is notified with an appropriate message.

Books Gallery and Detailed Book View

The books in the library is categorised in to different sections and displayed with a cover image. Clicking on the cover image will takes the user to a more detailed page where further information about the books is displayed.

Books Rentals

If a user is signed in, they are able to rent a book from the library.

Books Returns

The books rented by a User can be returned to the library.

Detailed User Profile and User Book management

Once the User logs in, the user is able to view the currently rented books by the user. The returned books by the user and the due dates for currently rented books can also be accessed from the user profile page. The rental and returns operations displays instant feedback on the user's profile page.

Search Functionality Case Sensitive

Using the search bar, anyone is able to search books. The search results will automatically filter the books by the title and author and display the results.

Admin Dashboard

A librarian can access the Library Dashboard, where all the trasaction records, currently rented books from the library, the Users registered in the library are displayed.

Admin Book Assign

A Librarian can assign a Books to a User by specifing the User's ID and the Books's ISBN

Admin Update Expiry Date

Librarians can update the expriry dates of the rented books by any user. Select the user from the Library Dashboard page and specify the number of days to increment or decrement the expiry date.
